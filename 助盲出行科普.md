## 📌 开篇 · 为什么需要这份 Skill

> 助盲出行科普是「高经验依赖、难以标准化、决策密度大」的传播任务。  
> 受众横跨从未关注视障群体的路人 → 视障者家属 & 公益从业者。平台、长度、视觉差异巨大，效果高度依赖选题策略、内容结构与发布节点。

## 🎯 模块1 · Skill 定义

> **一句话定义**  
> 用于「已有初步选题方向的助盲出行类科普项目」情境下，帮助内容策划与运营人员系统化解决「选题缺乏差异化、内容缺乏传播力、传播路径不清晰」三大难题，最终产出可在社交媒体以长图高效传播的科普内容。

### ✅ 适用场景
- 助盲出行垂直话题 · 单篇科普长图
- 传播载体：小红书/公众号/微博长图
- 用户类型：专业层（公益从业者/家属）+ 业余层（普通公众）

### ⚠️ 不适用场景
- 非长图形式的纯文字深度报道
- 多期连载大型专题策划
- 仅内部汇报不对外传播

### 关键假设
- **假设1**：话题具公共价值且无政策禁忌 → 失效表现：限流/下架
- **假设2**：制作人≥2-3天完整时间 → 失效：无法完成全流程
- **假设3**：具备数据抓取+文案能力 → 失效：先补能力

## ⭐ 模块3 · 关键判断点（价值占比40%）

> 每个判断点都是“真实执行中可能做错、且做错代价很大”的决策时刻。

### 🔍 判断点1 · 数据驱动型 vs 故事驱动型 【分类判断】

**触发情境**：选题锁定“助盲出行”，面临用权威数据引发理性共鸣，还是用真实故事引发情感共鸣。

**决策规则**：
- 数据驱动型（三者满足其二）：有权威冲击力数据 + 目标含专业层 + 首发深度区。
- 故事驱动型（三者满足其一）：有授权真实案例 + 受众为公众 + 计划发小红书/微博。
- ✅ 最优：**“数据开篇+故事主体”混合模式** — 前3屏数据建立信任，主体故事共鸣，结尾回归行动。

**反例警示**：没有明确受众就“两边都写一点”，导致专业层觉得不严谨，公众觉得枯燥。

---

### 🎯 判断点2 · 目标受众是哪一层？ 【分类判断】

**触发情境**：明确这份科普“到底说给谁听”。

**三层级策略**：
- 公众层 → 从零科普，强调认知差（如“3分钟盲道30处障碍”）
- 关注层 → 提供“我可以做什么”行动指南
- 专业层 → 深度数据+行业对比+国际视角

📌 **多层级覆盖**：公众号按“公众层（开头）→关注层（主体）→专业层（附录/深色卡片）”分层。

**反例**：把专业层内容直接推给公众 → 大量术语导致用户流失；或反过来被专业层评价“太浅”。

---

### 📏 判断点3 · 内容时长/长度如何取舍？ 【优先级判断】

**触发情境**：大纲阶段发现核心观点超过单条长图承载能力。

**决策规则**：
- 优先级：信息完整性 > 视觉美观 > 篇幅长度
- 核心信息点＞15 → 拆分为上下篇，单篇10-12个信息点
- 小红书 → 页数≤12页，每页文案≤80字
- 公众号 → 可放宽至20页以内，但前3页必须抓住读者
- 含专业层 → 附加“数据附录页”

**反例**：做30页长图，精华全埋在后面，读者到第5页就流失了。

---

### 💰 判断点4 · 预算有限时先投哪个平台？ 【优先级判断】

**触发情境**：制作+投流预算有限，无法覆盖所有平台。

**规则**：
- ≤500元制作+≤300元投流 → 主攻小红书（内容质量驱动）
- 500-1500元制作+300-800元投流 → 小红书 + 公众号转载
- ≥1500元 → 小红书+公众号+微博联动定制
- 目标专业影响力 → 主攻公众号+行业媒体，降低投流

**反例**：预算只够一个平台却强行“全网分发”，每个平台都没跑出效果。

---

### ⚠️ 判断点5 · 什么情况下需加入免责声明？ 【止损/升级】

**触发情境**：文案终审涉及：对具体组织批评、政策负面案例、未授权个人案例、医疗/技术建议。

**必须添加的情形**：
- 内容包含对机构/个人/政策批评
- 引用未经授权数据源
- 推荐第三方产品
- 未授权采访案例

**推荐添加**：制作主体为在校学生团队；涉及政策解读。

**位置**：长图末尾页，字号不小于正文70%。

**反例**：完全不做免责声明发出后因数据误差被指认，轻则删文道歉，重则引发舆论风波。

---

### 📉 判断点6 · 触发什么信号要调整传播策略？ 【止损/升级】

**触发情境**：发布后追踪数据，出现异常信号。

**信号与动作**：
| 信号 | 动作 |
|------|------|
| 冷启动失效（2h触达率低于历史50%） | 更换标题/封面图 |
| 流失率>60% | 优化首屏文案/视觉 |
| 互动率<2% | 增加文末引导或评论区经营 |
| 争议性评论 | 补发数据来源说明 |
| 内容被搬运 | 保留证据联系平台 |

**规则**：任一信号 → 24h内输出优化版；两项及以上启动B方案（改标题重发/故事版/KOL联动）。

**反例**：没改实质内容只换时间重发，重复打扰粉丝且无效。

## 📥📤 模块4 · 输入 / 输出标准

### 📌 必要输入
- 选题名称与描述（200字内）
- 目标受众定位（公众/关注/专业层）
- 核心数据来源（≥2个权威信源：中国盲协/住建部/统计局等）

### 📌 可选输入
- 制作预算范围
- 爆款科普案例链接
- 真实案例授权素材（降级：标注引用来源）

### ✅ 输出要求
- 长图(JPG/PNG) + 封面图 + 发布文案 + 转载稿
- 必含模块：首页痛点钩子｜数据区2-3组可视化｜案例区｜行动指南｜收尾署名+免责
- 风格：结论先行，单页信息量≤5条，小红书每页≤80字，微信≤120字，emoji/icon引导
- 验收标准：数据100%可溯源 + 自检7项达标 + 至少1个传播引爆点（数据冲击或真实共情）

## ⚠️ 模块6 · 风险点与优化

- **数据来源不可靠**：预警评论区质疑 → 所有数据交叉验证（中国盲协/住建部报告），引用新闻核对原始出处。
- **内容过于说教**：做完像教材 → Step2设置故事/案例卡，注入真实细节。
- **视觉过于拥挤**：单页文案上限80字（小红书友好），超出信息移到附录页。
- **传播节点踩空**：提前2天小范围预热，首发4小时内安排3-5位真实用户互动。
- **长图平台显示异常**：输出前按各平台裁剪预览，字号≥12pt，最高画质导出。

## 📡 附录 · 大模型分类 & 信源抓取指南

### 可用大模型
- 🇨🇳 国内：DeepSeek / 文心 / 通义 / Kimi / 智谱
- 🌍 国际：ChatGPT / Claude / Gemini / Grok

### 权威信源（AI必读）
- **政府/官方**：住建部、国家统计局、中国残疾人联合会 → 盲道覆盖率/无障碍进展
- **行业/协会**：中国盲人协会、中国残疾人福利基金会 → 视障人群统计
- **智库/研究**：民生智库、iTSTech → 国际经验对比
- **新闻/报道**：大众网、澎湃新闻 → 案例需溯源原始出处
- **学术研究**：CNKI、万方 → 区分结论与调研数据

**AI数据抓取规范**：每次输出“来源全称+发布日期+原始链接”；同一数据交叉验证；冲突时优先政府数据。
